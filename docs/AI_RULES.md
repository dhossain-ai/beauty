# AI_RULES

## Read Order
Before making changes, read:
1. MASTER_CONTEXT.md
2. STATUS.md
3. NEXT_PHASE.md
4. DECISIONS.md

## Core Execution Rules
- preserve the Soft Luxury Beauty Studio direction
- keep the project frontend-only
- keep content hardcoded
- prioritize mobile responsiveness
- prioritize premium visual quality
- prefer reusable components
- keep code clean and modular
- avoid unnecessary complexity
- do not introduce unrelated tools or architecture

## Do Not Introduce
- admin dashboard
- authentication
- database
- CMS
- backend APIs unless explicitly requested
- analytics dashboard
- booking engine backend
- unrelated pages
- large new dependency chains without good reason

## Design Rules
- elegant, premium, calm, image-led
- soft luxury palette
- strong spacing and typography hierarchy
- use motion carefully
- keep sections visually rich but not cluttered
- avoid generic template aesthetics
- avoid cheap decorative effects

## Responsive Rules
- mobile first in practice
- no cramped cards
- no broken hero on mobile
- no tiny unreadable text
- no desktop-only compositions
- sliders and interactive sections must remain usable on phone

## Change Rules
- do not redesign the whole project unless asked
- do not touch unrelated sections
- do not rename core project direction
- do not replace existing patterns without reason
- keep edits scoped to the active phase

## File Discipline
- avoid unnecessary new files
- avoid duplicate components
- avoid duplicate data structures
- keep docs concise
- update STATUS.md after meaningful progress
- update NEXT_PHASE.md when the active phase changes
- append major decisions to DECISIONS.md

## Delivery Rules
When a phase task is completed:
- summarize what was implemented
- mention any important design or technical choices
- mention anything intentionally deferred
- keep progress notes compact
