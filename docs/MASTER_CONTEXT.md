# MASTER_CONTEXT

## Project
Soft Luxury Beauty Studio

## Project Type
Frontend-heavy portfolio website for a women-focused beauty salon / beauty studio.
This is a premium marketing website concept built to look realistic, modern, sellable, and visually strong for portfolio use.

## Primary Goal
Build a high-end, mobile-responsive beauty salon website that feels image-led, elegant, polished, and conversion-focused.

## Secondary Goal
Create a reusable business website concept that could later be adapted for:
- beauty salons
- lash studios
- brow studios
- makeup studios
- nail studios
- skin / facial treatment studios
- premium local beauty businesses

## Current Scope
Frontend only.
No admin dashboard.
No authentication.
No backend.
No CMS.
No real booking system.
No database integration.

All content is hardcoded and structured cleanly so it can be converted later if needed.

## Core Product Positioning
A premium beauty studio website with:
- strong brand presentation
- service showcase
- specialist showcase
- gallery / before-after presentation
- blog / beauty journal
- clear booking CTA flow
- polished responsive experience

## Design Direction
Soft luxury.
Elegant, feminine, premium, modern, calm, image-led.

## Visual Tone
- warm white
- champagne beige
- soft blush
- taupe
- charcoal text
- light layered surfaces
- refined shadows
- premium editorial spacing

## UI Personality
- polished
- luxurious
- calm
- clean
- slightly editorial
- visually rich without clutter
- modern 2026 local business premium style

## Typography Direction
Use an elegant serif for major headings and a clean modern sans-serif for body and UI text.

The typography should feel premium and readable, not decorative or overly ornate.

## Layout Direction
- responsive first
- asymmetric but balanced sections
- strong image usage
- premium spacing
- clean content rhythm
- layered hero composition
- cards with soft radius
- subtle depth and overlap where useful

## Animation Direction
Animation should feel refined, soft, and premium.
Avoid noisy or over-animated behavior.

Preferred motion:
- fade-up reveal
- slight image zoom
- hover lift
- soft parallax only where safe
- smooth slider transitions
- staggered entrances
- subtle CTA emphasis

Avoid:
- heavy animation everywhere
- flashy effects
- jarring transforms
- long scroll-jacking sections
- excessive blur/glow

## Mobile Priority
Mobile responsiveness is mandatory.
The site must look premium on phone, tablet, and desktop.

Mobile goals:
- strong hero
- clean spacing
- swipe-friendly sections
- readable cards
- no cramped layouts
- simple menu
- easy CTA access

## Content Strategy
All content is hardcoded.
Use realistic salon-style content.

Content groups:
- services
- specialists
- testimonials
- gallery
- blog posts
- FAQ
- contact / location details
- pricing snippets

## Proposed Main Pages
- Home

## Homepage Goals
The homepage is the most important page.
It should immediately communicate:
- premium beauty brand
- trust
- services
- specialists
- results
- visual quality
- booking intent

## Homepage Key Sections
1. Navbar
2. Hero
3. Trust / review strip
4. Signature services
5. About / philosophy
6. Specialist showcase
7. Before and after / results section
8. Gallery preview
9. Testimonials
10. Blog preview
11. Final CTA
12. Footer

## Required Frontend Highlights
- luxury hero composition
- signature treatment slider / carousel
- before-after interactive section or strong comparison layout
- specialist portrait cards
- gallery mosaic or rich layout
- testimonial slider
- editorial blog cards
- strong responsive navbar and footer

## Slider / Interaction Priorities
Use only a few strong interactive areas, not too many.

Preferred:
- signature services carousel
- before / after comparison area
- testimonial slider
- swipe-friendly specialist cards on mobile

## Technical Direction
Recommended stack:
- Next.js
- TypeScript
- Tailwind CSS
- component-based structure
- static data files

## Static Data Suggestion
Use separate structured files for:
- services
- specialists
- testimonials
- gallery
- blog
- faq
- pricing
- site/contact info

## Constraints
- do not add backend
- do not add admin
- do not add auth
- do not overbuild
- do not turn this into a SaaS
- do not use generic cheap template styling
- do not make it too pink or childish
- do not make it feel like a medical clinic
- do not reduce the project to a plain minimal layout

## Success Criteria
The project is successful if it:
- looks premium and custom
- feels mobile-polished
- is visually richer than a standard local business site
- shows strong frontend skill
- is portfolio-ready
- could plausibly be sold as a beauty business website
